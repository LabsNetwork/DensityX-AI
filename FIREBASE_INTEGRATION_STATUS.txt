WHAT YOU NEED TO DO
================================================================================

STEP 1: Get Firebase Credentials (5 minutes)
─────────────────────────────────────────────
1. Go to https://console.firebase.google.com/
2. Select your project
3. Settings ⚙️ → Service Accounts → Generate New Private Key
4. Save the JSON file (keep it private and secure!)

STEP 2: Configure Backend (5 minutes)
──────────────────────────────────────
cd /Users/pratyush/git/DensityX-AI/backend

# Create .env from template
cp .env.example .env

# Edit .env and add your Firebase credentials:
# FIREBASE_CREDENTIALS_JSON='[paste entire JSON here]'
# FIREBASE_DATABASE_URL="https://your-project.firebaseio.com"
# FIREBASE_STORAGE_BUCKET="your-project.appspot.com"

nano .env  # or open in VS Code

STEP 3: Enable Firebase Services (2 minutes)
────────────────────────────────────────────
In Firebase Console:
1. Authentication → Enable Email/Password
2. Realtime Database → Create Database (test mode)
3. Storage → Create Bucket (optional)

STEP 4: Install & Run (5 minutes)
──────────────────────────────────
cd /Users/pratyush/git/DensityX-AI/backend

# Option A: Quick setup
bash firebase_quickstart.sh

# Option B: Manual setup
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Start server
python main.py

STEP 5: Verify (2 minutes)
───────────────────────────
Look for in logs:
[startup] Firebase connected successfully!

Then test:
curl -X POST http://localhost:8003/user/register \
  -H "Content-Type: application/json" \
  -d '{"ticket_id":"TEST","name":"User","phone":"9999"}'

Check Firebase Console → Realtime Database → users node for data

ENDPOINTS AVAILABLE

POST /user/register

  Input: {"ticket_id":"DX-001","latitude":13.085,"longitude":80.210,"gps_enabled":true}

  → Returns user profile

GET /user/active-users
  → Lists all registered users

  → Health check endpoint

   The project admin email can be different from your personal Gmail.

   Never commit to git, never share, never paste in chat.

5. ENVIRONMENT: Use .env files only in development. In production, set
   environment variables directly on the server/container.
