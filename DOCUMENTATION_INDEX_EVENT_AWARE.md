## 🔑 Key Concepts at a Glance

### Verified-Only Clustering
```
Only users with: verified==true AND gps_enabled==true
appear in clusters. Unverified users are filtered out.
```

### Adaptive Thresholds
```
Threshold changes based on:
- Event size (30% small → 15% stadium)
- Venue layout (tight × 0.80 → sprawling × 1.10)
- Event capacity (if approaching full)
```

### Risk Levels
```
Safe (0-49%)     🟢 Green → Monitor
Caution (50-99%) 🟠 Orange → Watch
Alert (100-150%) 🔴 Red-orange → Engage
Critical (150%+) 🔴 Red dashed → Full response
```

### Live Reshaping
```
Clusters automatically:
- Merge when users approach
- Split when users move apart
- Update centroids every 10 seconds
- Adjust sizes based on density
```

### Cluster Stability
```
Stability: 0.0 (new) → 1.0 (very stable)
- Increases with cluster age (stable at 2+ min)
- Increases with cluster size (stable at 50+ people)
- Used for smooth visual transitions
- Prevents noise from single updates
```
